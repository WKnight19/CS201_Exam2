A Balanced Tree
• A balanced tree in computer science refers to a type of
binary tree structure that maintains its height to be as low as
possible while elements are added or removed, ensuring that
operations like search, insert, and delete can be performed
efficiently.
• The primary goal of a balanced tree is to maintain its height
around 𝑂 log 𝑛 , where 𝑛 is the number of nodes.

Red-Black Trees
• A Red-Black Tree is a type of self-balancing binary search
tree.
• It maintains several additional properties to ensure that the
tree remains approximately balanced, which helps keep the
operations such as insertion, deletion, and searching
efficient, generally within 𝑂(log 𝑛) complexity, where 𝑛 is
the number of nodes in the tree.
Jiacheng Li, Ph.D.

Properties of red-black trees
A red-black tree is a binary search tree that satisfies the
following Red-black properties:
1. Every node is either red or black.
2. The root is black.
3. Every leaf (NIL) is black.
4. If a node is red, then both its children are black.
5. For each node, all simple paths from the node to descendant leaves
contain the same number of black nodes.
Proposition: The height of a red-black tree with n internal
nodes is ≤ 2 * lg(n+1) = O(lg n).

Red-Black Trees
• The rotation function in a Red-Black Tree is essential for
maintaining the tree's balance after insertions or deletions.
• In Red-Black Trees, the goal is to keep the tree's height
approximately logarithmic, which ensures that search, insert,
and delete operations can all be performed in O(lg 𝑛) time.
• Rotation functions:
➢A Left Rotation is applied when the tree becomes right-heavy (i.e.,
when a node has a heavy right subtree, causing the tree to become
unbalanced).
➢A Right Rotation is applied when the tree becomes left-heavy (i.e.,
when a node has a heavy left subtree).
