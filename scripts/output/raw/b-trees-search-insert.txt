Example - Opening a Large Image File
1. You click on a 2GB image file stored on your HDD/SSD.
2. The OS checks if the image is already in RAM (cache).
3. If not, the OS sends a DISK-READ request to the HDD/SSD.
4. The disk finds the file and loads a large portion of the image into
RAM.
5. Your image viewer software processes the image from RAM (not
directly from disk).
6. If you edit and save the file, the updated data is written back to disk
(DISK-WRITE).

• High Branching Factor Keeps Tree Shallow
➢Branching Factor: The number of children per node.
➢B-trees have a high branching factor (e.g., 50–2000 children per
node), keeping the tree shallow.
➢Shallow trees mean fewer disk reads.

DISK-READ()
• The DISK-READ operation is a function that deals with
reading data from a secondary storage device, like a hard
disk, into main memory.
• The primary purpose is to load a specific node of the B-tree
from disk into main memory for processing. In a B-tree used
in a database, the nodes are typically stored on disk,
and DISK-READ is necessary to interact with these nodes
during search, insert, or delete operations.
• In the context of B-trees, which are often used in databases
and file systems, this operation is crucial because B-trees can
be very large and not entirely fit into main memory.
Jiacheng Li, Ph.D.

Usage in B-Trees
• In the B-TREE-SEARCH algorithm, when the search for a
key descends from the current node to one of its children, the
child node may not be in main memory. The DISK-READ
operation is used to load the child node from disk into main
memory so that the search can continue.
• Typically, DISK-READ takes the address or identifier of the
node (often referred to as a pointer or a disk block address)
as a parameter. In the pseudocode, 𝑥. 𝑐𝑖 would be the
identifier for the child node 𝑖 of node 𝑥, where 𝑐𝑖 points to
the location on the disk where this child node is stored.
** It ensures that the data is brought into main memory.
Jiacheng Li, Ph.D.

Example
• When a B-tree node is accessed during a search and it is
