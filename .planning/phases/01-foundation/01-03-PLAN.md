    ```bash
    pnpm tsx --env-file=.env.local scripts/seed.ts
    ```

    - `b-trees.json`

    3. For each topic JSON:
    a. Insert into `topics` table using `db.insert(topics).values({ slug, title, description, order }).onConflictDoNothing()` -- conflict on the `slug` unique constraint
    b. Query the inserted/existing topic row to get its `id`
    c. Insert lessons using `db.insert(lessons).values([...lessons with topicId]).onConflictDoNothing()`

    4. The script MUST be idempotent per Pitfall 5. Re-running should not create duplicate rows. Use `onConflictDoNothing()` for both topics and lessons.

    5. Add a `lessons` unique constraint if not already present. The simplest approach: add a unique index on `(topicId, order)` to `lessons` table in schema.ts, then use `onConflictDoNothing()`. Alternatively, clear and re-insert (simpler for seed scripts): `await db.delete(lessons)` then `await db.delete(topics)` then insert fresh. Choose the clear-and-reinsert approach since this is a seed script and content is static.

    6. Log the count of inserted topics and lessons:
