## Typography

All sizes in px. Font: Inter (loaded via `next/font/google`).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.6 | Lesson concept prose; standard reading text |
| Label | 14px | 500 (medium) | 1.4 | Tab labels, badge text, nav items, metadata |
| Heading | 24px | 600 (semibold) | 1.2 | Topic page section headings (Concept, Pseudocode, C++ Code) |
| Display | 32px | 700 (bold) | 1.1 | Topic card titles on dashboard (`/`); page-level H1 |

Weights used: 400 (body) + 600 (headings). Medium (500) used only for UI labels — not for content.

Code blocks (pseudocode and C++ sections): `font-mono` at 14px, weight 400, line-height 1.7. Uses shadcn's default monospace stack (JetBrains Mono or system mono). Wrap in a `<pre>` with `overflow-x-auto`.

## Color

Theme: Zinc dark (shadcn Zinc palette, CSS variables). Values given as CSS variable names; actual hex resolved by shadcn init.

| Role | CSS Variable | Approximate Value | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `--background` | `#09090b` (zinc-950) | Page background, full-bleed surfaces |
| Secondary (30%) | `--card` | `#18181b` (zinc-900) | Topic cards, lesson content container, tab panel background |
| Accent (10%) | `--primary` | `#3b82f6` (blue-500) | Active tab indicator, "Sign in with Google" button, topic card hover ring |
| Destructive | `--destructive` | `#ef4444` (red-500) | Sign-out button (only destructive action in Phase 1) |
| Muted text | `--muted-foreground` | `#a1a1aa` (zinc-400) | Disabled tab labels, secondary metadata |
| Border | `--border` | `#27272a` (zinc-800) | Card borders, separator lines |

Accent (`--primary`) reserved for:
1. "Sign in with Google" button fill
2. Active tab underline indicator on `/topics/[topic]` page
3. Topic card hover ring / focus ring

No other elements use the accent color. Disabled tabs use `--muted-foreground` only.

## Page-Level Layout

### `/` — Dashboard

```
[Nav: Logo + User avatar/Sign-in]
[Page heading: "CS 201 — Exam 2 Topics" — Display/700]
[4-column grid on desktop, 2-column on tablet, 1-column on mobile]
  [Topic Card: Huffman Codes]
  [Topic Card: N-ary Trees]
  [Topic Card: Red-Black Trees]
  [Topic Card: B-Trees]
```

- Grid gap: `xl` (32px)
- Page horizontal padding: `lg` (24px) on mobile, `2xl` (48px) on desktop
- Topic card: `Card` with topic name as `CardHeader`, one-line description in `CardContent`, link wrapping the entire card

### `/auth/sign-in` — Sign-In

```
[Centered vertically and horizontally]
[Card: ~400px max-width]
  [Logo or app name]
  [Subheading: "Sign in to study"]
  [Button: "Sign in with Google" — full width, accent fill]
```

### `/topics/[topic]` — Topic Detail

```
[Nav: Logo + User avatar + Sign-out]
[Topic heading: Display/700]
[Tabs: Lesson | Visualizations | Quiz | Flashcards]
  [TabsContent: Lesson (active)]
    [Section: Concept — <h2> + prose <p>]
    [Section: Pseudocode — <h2> + <pre><code>]
    [Section: C++ Implementation — <h2> + <pre><code>]
  [TabsContent: Visualizations — disabled, greyed]
  [TabsContent: Quiz — disabled, greyed]
  [TabsContent: Flashcards — disabled, greyed]
```

- Lesson content max-width: `768px` (prose width), centered
- Section spacing between anchored sections: `2xl` (48px)

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Primary CTA | "Sign in with Google" | AUTH-01 — one click, no additional fields; source decision D-08 |
| Dashboard page heading | "CS 201 — Exam 2 Topics" | Descriptive, scope-clear |
| Dashboard subheading | "Select a topic to start studying" | Empty guidance copy |
| Topic card CTA | "Open topic" (SR-only; full card is linked) | Accessibility label only |
| Empty lesson state | "Content loading…" (skeleton) | Lesson seeded at DB seed time — should never be empty post-seed |
| Auth error state | "Sign-in failed. Try again or check your Google account settings." | Google OAuth failure path |
| Session expired state | "Your session has expired. Sign in again to continue." | Session timeout copy |
| Disabled tab tooltip | "Available in a future update" | Phase 1 scope — Visualizations / Quiz / Flashcards tabs disabled |
| Sign-out confirmation | None — sign-out is reversible (instant re-auth via Google), no confirmation modal needed | D-08 notes sign-out redirects to sign-in screen |
| Nav sign-in prompt | "Sign in" | Pre-auth nav state |

No destructive data actions in Phase 1 (no delete operations, no user-created content). Sign-out is the only action with irreversible session consequence — handled without a confirmation modal since re-auth is one click.

## Interaction Contracts

### Sign-In Flow
1. Unauthenticated user visits any route → redirect to `/auth/sign-in`
2. User clicks "Sign in with Google" → button shows spinner, text becomes "Signing in…"
3. Google OAuth redirect → callback → Better Auth creates session → redirect to `/` dashboard
4. Failed OAuth → return to `/auth/sign-in` with error message below button

### Topic Navigation
1. Dashboard shows 4 topic cards — entire card surface is clickable (no button-within-card pattern)
2. Clicking a topic card navigates to `/topics/[topic]`
3. Topic page opens with Lesson tab active by default
4. Disabled tabs (Visualizations, Quiz, Flashcards): visible, pointer-events-none, `opacity-50`, tooltip "Available in a future update" on hover

### Loading States
- Dashboard topic cards: show `Skeleton` placeholder (same card dimensions) during data fetch
- Lesson content: show `Skeleton` lines matching approximate prose layout during SSR
- Sign-in button: spinner replaces icon during OAuth redirect, button disabled
