  <name>Task 1: Create Docker infrastructure and enable standalone output</name>
  <files>.dockerignore, Dockerfile, docker-compose.yml, next.config.ts</files>
  <action>
**1. Create `.dockerignore`:**

Exclude build artifacts and non-essential files from Docker context:
```
node_modules
.next
.git
.gitignore
.planning
*.md
.env*
.vercel
coverage
.pnpm-debug.log*
```
Keep `.env*` out of context — env vars are injected at runtime via docker-compose `env_file`.

**2. Create `Dockerfile` with multi-stage build:**

Use `node:24-alpine` as base. Enable corepack for pnpm 9.

Stages:
- **base**: `node:24-alpine`, `corepack enable`, `corepack prepare pnpm@9 --activate`
- **deps**: FROM base, WORKDIR /app, COPY `package.json` + `pnpm-lock.yaml`, run `pnpm install --frozen-lockfile`. This layer caches unless lockfile changes.
- **dev** (target for dev compose): FROM deps, COPY entire source, EXPOSE 3000, CMD `pnpm dev`. Source is volume-mounted in compose so the COPY is just a fallback.
- **builder**: FROM deps, COPY entire source, run `pnpm build`. Produces `.next/standalone` because of the config change below.
- **runner** (target for prod compose): FROM `node:24-alpine` (clean image, no pnpm), WORKDIR /app, COPY `--from=builder /app/.next/standalone ./`, COPY `--from=builder /app/.next/static ./.next/static`, COPY `--from=builder /app/public ./public` (if public/ exists — check first, skip if not). EXPOSE 3000, ENV `PORT=3000`, ENV `HOSTNAME="0.0.0.0"`, CMD `["node", "server.js"]`.

Key details:
- The dev target does NOT run `next build` — it runs `next dev` for hot reload
- The runner target is ~150MB (alpine + standalone) vs ~1GB+ (full node_modules)
- `HOSTNAME="0.0.0.0"` is required for Next.js to listen on all interfaces inside container

**3. Create `docker-compose.yml`:**

```yaml
services:
  dev:
    build:
      context: .
      dockerfile: Dockerfile
      target: dev
    ports:
      - "3000:3000"
    volumes:
      - .:/app
      - /app/node_modules
    env_file:
      - .env.local
    profiles:
      - dev

  prod:
    build:
      context: .
      dockerfile: Dockerfile
      target: runner
    ports:
      - "3000:3000"
    env_file:
      - .env.local
    profiles:
      - prod
```

Volume mount notes:
- `.:/app` gives hot reload — source changes reflect immediately
- `/app/node_modules` anonymous volume prevents host node_modules from shadowing container's installed deps
- No database service — Neon is external, connection string comes from DATABASE_URL in .env.local

Profiles keep dev and prod separate: `docker compose --profile dev up` or `docker compose --profile prod up`.

**4. Update `next.config.ts`:**

Add `output: "standalone"` to nextConfig:
```typescript
const nextConfig: NextConfig = {
  output: "standalone",
};
```

This tells Next.js to produce a self-contained server in `.next/standalone/` that includes only the node_modules files actually needed at runtime. Required for the production Docker stage.

<task type="auto">

**1. Build the dev image:**
```bash

```
Confirm it completes without errors. The dev image should install deps via pnpm.

**2. Build the prod image:**
```bash
docker compose --profile prod build
```
This runs `pnpm build` inside the builder stage and produces the standalone output. Confirm it completes without errors.

**3. Smoke-test dev container:**
```bash
docker compose --profile dev up -d
```
Wait 10-15 seconds for Next.js dev server startup, then:
```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000
```
Expect 200 or 307 (redirect to login is fine — proves server is running).
Then tear down: `docker compose --profile dev down`

**4. Smoke-test prod container:**
```bash
docker compose --profile prod up -d
```
Wait 5 seconds (standalone starts faster), then:
```bash
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000
```
Expect 200 or 307. Then tear down: `docker compose --profile prod down`

**5. Verify image sizes:**
```bash
docker images | grep cs201
```
Dev image will be larger (~800MB+ with full node_modules). Prod image should be significantly smaller (~200-300MB with standalone).

If builds fail due to missing `public/` directory, remove the `COPY public` line from the runner stage — the app may not have a public folder yet.
