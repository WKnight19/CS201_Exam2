<task type="auto">
  <name>Task 1: Fix Dockerfile, .dockerignore, and docker-compose.yml</name>
  <files>Dockerfile, .dockerignore, docker-compose.yml</files>
  <action>
**Dockerfile** — rewrite with these fixes applied to the existing multi-stage structure:

1. **base stage** (node:24-alpine): Enable corepack, prepare pnpm@9. Unchanged from current.

2. **deps stage**: Copy package.json, pnpm-lock.yaml, AND pnpm-workspace.yaml (pnpm 9 needs it for `ignoredBuiltDependencies`). Run `pnpm install --frozen-lockfile`.

3. **dev stage** (FROM deps): Copy all source. Expose 3000. CMD `pnpm dev`. No changes needed.

4. **builder stage** (FROM deps): Copy all source. Run `pnpm build`. No changes needed.

5. **runner stage** (FROM node:24-alpine) — FIX these issues:
   - Add non-root user: `RUN addgroup --system --gid 1001 nodejs && adduser --system --uid 1001 nextjs`
   - Set `WORKDIR /app`
   - Set `ENV NODE_ENV=production`, `ENV PORT=3000`, `ENV HOSTNAME="0.0.0.0"`
   - **BUG FIX**: Add `COPY --from=builder /app/public ./public` — currently missing, so static assets (SVGs) won't load in production
   - Copy standalone: `COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./`
   - Copy static: `COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static`
   - `USER nextjs`
   - `EXPOSE 3000`
   - `CMD ["node", "server.js"]`

**.dockerignore** — keep current entries, add:
- `Dockerfile` and `docker-compose.yml` (no reason to send these into build context)
- `*.log` (catch any log files)
- Remove `pnpm-workspace.yaml` from ignore list (it was added previously but pnpm needs it during install)

**docker-compose.yml** — update both services:

Dev service:
- Add `.next` anonymous volume (`/app/.next`) to prevent hot-reload .next directory conflicts between host and container
- Keep existing bind mount (`.:/app`) and node_modules anonymous volume

Prod service:
- Add `restart: unless-stopped`
- Add healthcheck: `curl -f http://localhost:3000/ || exit 1` with interval 30s, timeout 10s, retries 3, start_period 10s
- Install curl in runner stage for healthcheck (`RUN apk add --no-cache curl` before USER directive) OR use wget which is already in alpine (`wget -q --spider http://localhost:3000/`)

Both services keep their profile-based activation (`profiles: [dev]` / `profiles: [prod]`).
